# 大额消费可行性评估 设计文档

**日期**：2026-06-20
**针对 Skill**：`family-financial`（family-financial-skill）

## 1. 目标

用户基于自己的家庭财务状态咨询「某笔大额消费是否扛得住」。技能根据资产负债/现金流/各项指标，评估这笔花销的冲击，给出 **可承受 / 谨慎 / 暂不建议** 的判定 + 关键数字 + 理由 + 可落地的替代方案。

属于**消费预算分析**（基于用户自身数据），**不是投资建议**——不推荐具体标的、不做择时，延续现有「非持牌顾问」免责。

## 2. 已确认的设计决策

| 决策点 | 选择 |
|---|---|
| 核心场景 | 大额消费可行性评估（一次性能力检查） |
| 付款方式 | 同时支持 一次性全款 / 分期（贷款） |
| 判定口径 | 沿用现有健康指标（应急储备 3–6 个月、偿债收入比 ≤40%） |
| 执行位置 | 确定性计算进脚本（可单测）；自然语言解析与建议措辞在 agent 层 |

## 3. 架构

- **脚本**：新增纯函数 `affordability(...)` + CLI 子命令 `afford`，零联网、零依赖、可单测。
- **Agent（SKILL.md）**：把「想买 30 万的车，分期 36 期」解析为参数，调 `afford` 拿判定与数字，再用通用理财常识包装为建议。建议不含投资推荐。

## 4. 输入与计算

### 4.1 输入
- `amount`：消费金额（¥，正数）
- `mode`：`lump`（一次性）/ `installment`（分期）
- 分期补充：`monthly`（月供）或 `months`（期数）。只给 `months` 时按**无息估算** `monthly = amount / months`。

### 4.2 从快照/流水算出的现状量
- 月收入、月支出、月结余（当月 income_statement）
- 流动资产合计（snap 中 流动性=流动 的资产）
- 现有月还债额（当月 现金流分类=筹资 且 方向=流出 合计）
- 总资产、总负债、负债率（balance_sheet）

### 4.3 冲击量
- **一次性**：付款来源默认流动资产 →
  - 流动资产剩余 = 流动资产 − amount
  - 付后应急储备(月) = (流动资产 − amount) / 月支出
- **分期**：
  - 新偿债收入比 = (现有月还债额 + monthly) / 月收入
  - 新月结余 = 月结余 − monthly
  - 负债率变化：新负债率 = (总负债 + amount) / 总资产（分期视为新增等额负债，粗略）

## 5. 判定规则

### 5.1 一次性全款（金额 A，流动资产 L，月支出 E）
- **暂不建议**：A > L（流动资产不足以支付），或 (L − A)/E < 3（付后应急储备 < 3 个月）
- **谨慎**：3 ≤ (L − A)/E < 6
- **可承受**：(L − A)/E ≥ 6

### 5.2 分期（新增月供 M，月收入 I，现有月还债 D，月结余 S）
- **暂不建议**：S − M < 0（月供吃掉结余），或 (D + M)/I > 0.4
- **谨慎**：0.3 < (D + M)/I ≤ 0.4
- **可承受**：(D + M)/I ≤ 0.3 且 S − M > 0

### 5.3 无法评估
- 一次性且月支出为 0、或分期且月收入为 0 → 返回判定 `无法评估` + 原因，提示补数据，不臆断。

## 6. 临界值（可落地的替代方案）

让建议可执行，脚本额外算「最多能花多少仍可承受」：
- **一次性**：`max_affordable = max(0, 流动资产 − 6 × 月支出)`，文案「降到 ¥X 以内为可承受」。
- **分期**：`max_monthly = max(0, 0.3 × 月收入 − 现有月还债)`，文案「月供控制在 ¥Y 以内为可承受」；若给了期数，可换算 `max_amount ≈ max_monthly × 期数`。

## 7. 输出结构

`affordability(...)` 返回 dict：
- `判定`：可承受 / 谨慎 / 暂不建议 / 无法评估
- `指标`：付后应急储备 或 新偿债收入比、新月结余、负债率变化 等（按 mode）
- `临界值`：max_affordable（lump）或 max_monthly（installment）
- `理由`：list[str]，由规则生成的事实性说明

报表/对话由 agent 渲染为：判定 + 关键数字 + 理由 + 替代方案 + 免责。

## 8. CLI

```
python3 scripts/family_finance.py afford --amount 300000 --mode lump --data-dir .
python3 scripts/family_finance.py afford --amount 300000 --mode installment --months 36 --data-dir .
python3 scripts/family_finance.py afford --amount 300000 --mode installment --monthly 9000 --data-dir .
```
默认报告基于「最近快照 + 当月流水」。可加 `--month YYYY-MM`（默认当月/最近月）。

## 9. SKILL.md 工作流

用户说「想买 30 万的车，合适吗 / 分期 36 期 / 月供 9000」时：
1. 解析 amount、mode、monthly/months（不明确则向用户确认，例如「全款还是分期？」）。
2. 调 `afford` 取判定与数字。
3. 输出：判定 + 关键数字 + 理由 + 替代方案（如「降到 ¥X 可承受」「改 48 期月供降到 ¥Y」）。
4. 免责：基于你自身数据的**消费预算分析**，非投资建议、非持牌顾问。
5. 无法评估时说明原因、请补数据。

## 10. 测试

- 一次性：可承受 / 谨慎 / 暂不建议（应急 ≥6 / 3–6 / <3）三档；流动资产不足（A>L）；月支出为 0 → 无法评估。
- 分期：可承受 / 谨慎 / 暂不建议（偿债 ≤0.3 / 0.3–0.4 / >0.4）；月供吃掉结余（S−M<0）；月收入为 0 → 无法评估；只给期数时无息估月供。
- 临界值：max_affordable / max_monthly 计算正确（含负数归零）。

## 11. 范围之外（YAGNI）

- 利率/IRR 精算（分期默认无息估）
- 多笔消费组合、跨月预算预测
- 月度可支配预算、省钱优化建议（后续可作为独立特性）
- 投资类建议（明确不做）
