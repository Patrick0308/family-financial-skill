# 资产价格评估能力 设计文档

**日期**：2026-06-20
**针对 Skill**：`family-financial`（family-financial-skill）
**作者署名**：白泽 / Patrick

## 1. 目标

让用户录入**持仓**（股票/基金）和**房产**信息后，技能尽力查询当前市场行情/均价，给出人民币计价的**估值**，并写入资产负债快照，从而让净资产、健康评分、财富等级反映接近市价的真实状况。

核心约束：股票/基金估值可靠（实时行情），房产估值只能粗估（区域均价 × 面积），二者**置信度必须区分并诚实标注**。

## 2. 已确认的设计决策

| 决策点 | 选择 |
|---|---|
| 覆盖资产 | 股票/基金 + 房产，**分置信度** |
| 估值执行位置 | **Agent 层联网查，Python 脚本保持纯净不联网** |
| 记录方式 | 存**输入**（持仓代码+份额 / 房产小区+面积），可反复重估 |
| 外币 | **多市场，自动按汇率折算人民币**（A股/港股/美股） |
| 持仓刷新 | 支持「刷新估值」一键按最新价重算 |

## 3. 架构

延续既有原则：**确定性计算在 Python 标准库脚本里，联网/查询能力在 agent（Claude）层**。

- Agent 负责：解析自然语言、查行情、查汇率、搜房产均价、折算人民币、判断置信度、写 CSV。
- 脚本负责：读 CSV（含新增可选列）→ 计算三表/评分/等级 → 渲染报表（含估值标注）。脚本**不联网、零第三方依赖、可单测**，这一点不变。

## 4. 数据文件

数据目录新增两个「输入清单」文件；资产负债快照 `balances.csv` 由它们派生 + 用户手填项组成。

```
数据目录/
├── holdings.csv      # 股票/基金持仓（新增）
├── properties.csv    # 房产（新增）
├── balances.csv      # 资产负债快照（沿用，新增 3 个可选列）
├── transactions.csv  # 流水（沿用，不变）
└── reports/
```

### 4.1 holdings.csv

| 列 | 说明 | 例 |
|---|---|---|
| 代码 | 行情代码（带市场后缀） | `00700.HK` |
| 名称 | 人类可读名 | `腾讯控股` |
| 市场 | A股 / 港股 / 美股 | `港股` |
| 份额 | 持有股数/份额 | `500` |
| 资产分类 | 写入快照时的项目归类（默认「投资」，标签 流动/可投资） | `投资` |
| 备注 | 可选 | `长期持有` |

代码后缀约定：A股 `600519.SS`（沪）/ `000001.SZ`（深）；港股 `00700.HK`；美股 `AAPL`（无后缀）。

### 4.2 properties.csv

| 列 | 说明 | 例 |
|---|---|---|
| 项目名 | 写入快照时的资产项目名 | `自住房` |
| 小区或区域 | 估值搜索粒度（**不存详细门牌**） | `深圳南山某小区` |
| 面积㎡ | 建筑面积 | `89` |
| 性质 | 自住 / 投资 | `自住` |
| 备注 | 可选 | `2020 年购入` |

性质→标签映射：自住 = 非流动·自用；投资 = 非流动·可投资。

### 4.3 balances.csv 新增可选列

原列：`日期, 类型, 项目, 金额, 流动性, 性质`
新增 3 个**可选**列（旧数据留空仍兼容）：

| 新列 | 说明 | 取值 |
|---|---|---|
| 来源 | 数值怎么来的 | `行情` / `搜索` / `手填`（空=手填） |
| 估值日期 | 该数字对应的市场日期 | `2026-06-20` |
| 置信度 | 可靠程度 | `高` / `中` / `低`（空=不标注） |

脚本 `load_balances` 用 `.get(..., "")` 读取这三列，不破坏旧数据。

## 5. 工作流（自然语言触发）

| 用户说 | Agent 行为 |
|---|---|
| 「加持仓：腾讯 500 股」 | 解析代码/市场（不确定则确认），追加到 holdings.csv |
| 「房子：南山 XX 小区 89 平，自住」 | 追加到 properties.csv |
| 「刷新估值」/「更新余额」 | ① 每个持仓查现价+汇率→折人民币市值；② 每套房联网搜区域均价×面积→粗估；③ 连同用户口头报的现金/存款/负债，写成**同一日期**的快照到 balances.csv（含 来源/估值日期/置信度 列） |
| 「出 X 月的表」 | 照常出表；报表标注估算项、汇率、估值日期 |

**估值换算口径**：人民币市值 = 原币现价 × 份额 × 汇率。A股汇率=1。港股取 HKDCNY，美股取 USDCNY（agent 用行情/搜索取当日汇率）。

**快照构成**：一次「更新余额」的快照行 = agent 派生项（股票/房产）+ 用户手填项（现金、存款、负债等）。共用同一快照日期。

## 6. 报表标注

`render_report` 对带「来源/置信度」的资产行追加小标注，让估算与手填一目了然：

> 投资房 ¥2,000,000 ⚠估（搜索·2026-06-18·低）
> 腾讯控股 ¥165,400（行情·2026-06-20）

并在报表免责区追加一句：含市场估值的项目为某时点估算，房产为区域均价粗估，仅供参考。

脚本侧改动仅为「多读 3 列 + 渲染时多拼一个标注字符串」，仍然零联网、确定性、可单测。

## 7. 错误处理

- **代码不明/查不到**：agent 向用户确认（哪只、哪个市场），绝不臆填。
- **网络/行情失败**：保留上次市值，标 `置信度=低` + 旧 `估值日期`，并明确告知本次未刷新成功。
- **房产**：永远 `置信度=低`，给估算区间 + 均价出处链接；报表免责重申房产为粗估。

## 8. 隐私与边界

- `holdings.csv`、`properties.csv` 含持仓与住址，属敏感数据：加入 `.gitignore`，**绝不入库**（与 transactions.csv/balances.csv 一致）。
- 联网搜索只发**小区/区域**粒度，不发详细门牌。
- 估值仅为「按市价给持仓计值」，**不构成投资建议**（不推荐买卖、不做择时），延续现有「非持牌顾问」免责。

## 9. 范围之外（YAGNI）

- 自动定时刷新、历史价格曲线、收益率/盈亏分析
- 接入券商/银行账户自动同步
- 房产 AVM 级精确估值（无可靠免费数据源）
- 多币种账本（账本仍单币种人民币，仅在估值时折算）

4. `.gitignore`：忽略根目录 `holdings.csv`、`properties.csv`。

6. 全程**不**给脚本引入网络或第三方依赖；估值数据由 agent 写入 CSV。
