# 分期评估增强：现金缓冲 + 首付 设计文档

**日期**：2026-06-20
**针对 Skill**：`family-financial`
**前置**：在已实现的 `affordability()`（见 `2026-06-20-affordability-advice-design.md`）基础上增强分期分支。

## 1. 背景与目标

v1 的分期评估只看「还款面」（偿债比 + 月结余），忽略了现金：① 现实分期常需**首付**（一次性现金支出）；② 背上固定月供后更需要**现金缓冲**，万一降薪/失业还得继续还。本增强把现金重新纳入分期判定，并支持「首付 + 分期」混合。

## 2. 已确认的设计决策

| 决策点 | 选择 |
|---|---|
| 现金缓冲口径 | 复用应急储备：付完首付后 (流动资产−首付)/月支出，沿用 3–6 个月线 |
| 首付输入 | `--amount` 为总价，新增 `--down`（默认 0）；分期金额 = 总价 − 首付 |
| 判定合成 | 现金面 A 与 还款面 B **取更严的一档** |
| 全款分支 | 不变 |

## 3. 改动点

- `affordability(snap, txns, amount, mode, monthly=None, months=None, down=0)`：新增 `down` 参数；重写 installment 分支。
- CLI `afford`：新增 `--down`（float，默认 0）。
- lump 分支与现有 API 其余部分不变；纯分期（down=0）会因新增「现金面 A」而比 v1 更严（这是预期的盲区修补）。

## 4. 分期判定逻辑

记：流动资产 L、月支出 E、月收入 I、现有月还债 D（筹资类流出合计）、月结余 S（=收入−支出）、首付 down、总价 amount。

- 分期金额 financed = amount − down
- 月供 M = monthly（若给）否则 financed / months（无息估）

**硬性「暂不建议」**（任一成立即判，理由明确）：
- down > L（连首付都掏不出）
- S − M < 0（月供吃掉结余，月光）

**否则取 A、B 更严的一档：**

- **A 现金面**：post_reserve = (L − down) / E
  - ≥6 → 可承受；3 ≤ · < 6 → 谨慎；< 3 → 暂不建议
- **B 还款面**：new_ratio = (D + M) / I
  - ≤0.3 → 可承受；0.3 < · ≤ 0.4 → 谨慎；> 0.4 → 暂不建议

档位序：可承受 < 谨慎 < 暂不建议；**最终判定 = max(A 档, B 档)**。理由点明被现金面还是还款面卡住（并列时两者都说）。

## 5. 临界值（可落地）

- **可承受月供上限** = max(0, 0.3 × I − D)（还款面回到 ≤30%）
- **首付上限**（保留 ≥3 个月应急）= max(0, L − 3 × E)（现金面回到 ≥3 个月）

## 6. 无法评估 / 边界

- I ≤ 0 或 E ≤ 0 → 无法评估（A 需 E、B 需 I，缺一不可），理由提示补记录。
- down ≥ amount → 无法评估，理由「首付已覆盖全价，请用全款（lump）评估」。
- monthly 未给且 months 缺失/≤0 → 无法评估（沿用 v1）。

## 7. 输出结构

installment 返回 dict（沿用 判定/指标/临界值/理由）：
- `指标`：付后应急储备、月供、新偿债收入比、新月结余、新负债率
- `临界值`：可承受月供上限、首付上限
- `理由`：list[str]，点明绑定约束（现金面/还款面/硬性条件）

lump 分支输出不变。

## 8. 测试

- 还款面卡住（现金充裕但偿债比 >40%）→ 暂不建议（理由含还款面）。
- 现金面卡住（偿债比很低但付完首付后应急 <3 个月）→ 暂不建议（理由含现金面）。
- 取更严：A=谨慎、B=可承受 → 整体谨慎。
- 首付掏不出（down > L）→ 暂不建议。
- 月光（S − M < 0）→ 暂不建议。
- 含首付混合：down>0，financed 与 M 正确，A/B 都算对。
- 首付 ≥ 总价 → 无法评估（提示用全款）。
- 缺收入 / 缺支出 → 无法评估。
- 临界值：可承受月供上限、首付上限计算正确（含负数归零）。
- 向后兼容：down 默认 0 时，旧 CLI 调用与既有分期测试仍可用（但纯分期现在会叠加 A 检查——相应更新受影响的既有分期测试，使其数据同时满足 A 才能保持原判定）。

## 9. 范围之外（YAGNI）

- 利率 / IRR 精算（分期仍按无息估月供）
- 「最优首付/期数」求解（只给上限，不做优化）
- 多笔消费组合
